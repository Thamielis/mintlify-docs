## Overview

Use this section to document each PowerShell project you maintain.

## Recommended structure

For each project, include:

1. A short summary of what the project does.
2. Installation and setup steps.
3. Usage examples with real commands.
4. Configuration options and defaults.
5. Troubleshooting and known issues.

## Add your first project page

1. Copy `powershell-projects/project-template.mdx`.
2. Rename it to your project name.
3. Add it to the PowerShell navigation in `docs.json`.
4. Fill in each section with project-specific details.
