---
title: "Release checklist"
description: "Checklist for shipping PowerShell project updates."
---

## Pre-release

- [ ] Update module version.
- [ ] Update changelog entries.
- [ ] Validate examples still work.
- [ ] Confirm compatibility notes are current.

## Validation

- [ ] Run tests locally.
- [ ] Run linting and formatting checks.
- [ ] Verify install instructions on a clean environment.
- [ ] Verify docs links are not broken.

## Publish

- [ ] Create release notes.
- [ ] Publish module/package.
- [ ] Tag the release in source control.
- [ ] Announce the release with upgrade notes.

## Post-release

- [ ] Confirm install from public feed works.
- [ ] Confirm telemetry or usage checks are healthy.
- [ ] Open follow-up items for known gaps.
